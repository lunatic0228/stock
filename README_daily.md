### 建議執行時間

| 時間 | 用途 |
|---|---|
| 早上 8:30（開盤前） | 看昨日收盤訊號，決定今天是否掛單 |
| 下午 1:00（盤中） | 快速確認有沒有觸發緊急警示，還有 30 分鐘可以處理 |
| 下午 2:30（收盤後） | 看今日最終收盤訊號，規劃明日操作 |

> 程式會自動判斷盤中 / 盤後，顯示對應提示。
> 盤中：MA5/MA10 保持前一日穩定值，量比為累計量推估全日，僅供參考。
> 盤後：MA5/MA10 以 Fugle 最終收盤價重算，精確對齊 Yahoo Finance / TradingView。

## 設定持股（stocks.py）

打開 `stocks.py` 修改以下兩個區塊：

### 持倉設定

```python
HOLDINGS = {
    "2330.TW": {
        "name":      "台積電",        # 公司名稱（顯示用）
        "buy_price": 950.0,           # 平均買入價
        "shares":    1000,            # 持有股數
        "avg_down":  False,           # True = 深套，找低點攤平
        "building":  True,            # True = 建倉中，持續買進
    },
    "NVDA": {
        "name":      "輝達",
        "buy_price": 120.0,
        "shares":    10,
        "avg_down":  False,
        "building":  True,
    },
}
```

**avg_down 和 building 的差別：**

| 旗標 | 情境 | 程式行為 |
|---|---|---|
| `avg_down: True` | 深套，等反彈低接 | 偵測超賣反彈訊號（4個條件） |
| `building: True` | 持續建倉，順勢加碼 | 偵測順勢加碼時機（4個條件） |
| 兩個都 True | 深套但仍想持續加碼 | 同時顯示兩種訊號 |

### 觀察名單（還沒買的股票）

```python
WATCHLIST = {
    "tw": ["2330", "2317"],   # 台股代號（不用加 .TW）
    "us": ["NVDA", "AAPL"],   # 美股代號
}
```

### 更新持股均價（加碼後）

加碼後記得更新 `buy_price` 和 `shares`，例如：
```python
# 原本：100 股，均價 80 元
# 加碼：再買 100 股，成交 75 元
# 新均價：(80×100 + 75×100) / 200 = 77.5 元

"buy_price": 77.5,
"shares":    200,
```

## 報告說明

### 區塊一：持倉警示

每檔持股顯示現價、損益，以及以下警示：

**ATR 停損線（三段式，依獲利自動切換）：**

| 獲利狀況 | 停損公式 | 標籤 | 說明 |
|---|---|---|---|
| 虧損 / 獲利 < 10% | 買入價 − 2×ATR | 初始停損 | 保護本金 |
| 獲利 10~20% | 買入價 + 獲利×40% | 保利停損 | 鎖住至少 40% 獲利 |
| 獲利 ≥ 20% | **現價 − 2×ATR**（追蹤） | 移動停損 | 跟著現價走，鎖住大部分利潤 |

**均線跌破訊號：**

| 符號 | 訊號 | 建議動作 |
|---|---|---|
| 🔴 | 同時跌破 MA5 + MA10 | 趨勢轉弱，建議減碼或出場 |
| 🔴 | 單日跌幅 > 4% | 注意跳空，考慮減碼 |
| 🔴 | 跌破 ATR 停損線 | 建議出場 |
| 🟠 | 單獨跌破 MA10 | 中期支撐失守，考慮減碼 30% |
| 🟠 | 跌破 MA5 且 MACD 翻負 | 短線轉弱，考慮部分出場 |
| 🟡 | 跌破 MA5 但 MACD 仍正 | 觀望，留意明日走勢 |

**停利訊號（乖離率 + RSI 雙重確認）：**

| 符號 | 條件 | 建議動作 |
|---|---|---|
| 🟢 | 乖離率 > 12% 且 RSI > 75 | 強力停利，建議減碼 30~50% |
| 🟢 | 乖離率 > 8% 且 RSI > 68 | 過熱，建議先減碼 30% |
| 🟡 | 乖離率 > 5% 且 RSI > 62 | 偏熱，留意是否獲利了結 |
| ✅ | 無任何警示 | 續抱 |

**基本面資料說明（近12個月滾動，僅供參考）：**
- 本益比、本淨比、殖利率：來自台灣證交所/櫃買中心官方 API，每日更新
- 月營收 YoY/MoM：來自 FinMind，為實際公布月份資料（台股月營收每月 10 日公布）

### 區塊二：攤平時機偵測（avg_down: True 的持股）

**盤中、盤後均會顯示。** 4 個條件，達到 3 個以上顯示「攤平訊號確認」：

| 條件 | 說明 |
|---|---|
| RSI < 60 且持續回升 | 賣壓釋放，止跌訊號；RSI ≥ 60 表示已過高不適合攤平 |
| MA5 斜率翻正 | 短線止跌跡象 |
| 今日收紅且量比 > 1.2 | 有實際買盤進場 |
| 距近20日低點反彈 > 3% | 底部有支撐，確認低點 |

> RSI 採用 Wilder's SMMA 計算（與 Yahoo Finance / TradingView 一致），門檻設 60 而非 35，避免因演算法精準度提升導致條件過嚴。

### 區塊三：建倉加碼機會（building: True 的持股）

4 個條件全達成才顯示「加碼時機」：

**路徑 A — 回調加碼（預設，較安全）：**

| 條件 | 說明 |
|---|---|
| MA5 > MA10 | 多頭排列，方向對 |
| RSI 40~65 | 有動能但不追高（>65 過熱，<40 動能弱） |
| 量比 ≥ 0.5 | 量縮回調可接受（不追量大高點） |
| 現價距 MA5 ≤ 5% | 回調至均線附近，不追高 |

**路徑 B — 突破確認（次選，追強）：**

當以下 4 個條件同時成立時，直接觸發：

| 條件 | 說明 |
|---|---|
| MA5 > MA10 | 多頭排列 |
| RSI 50~75 | 強勢但未過熱 |
| 量比 ≥ 1.5 | 放量突破 |
| 日漲幅 ≥ 3% | 明確突破訊號 |

### 區塊四：明日進場機會（WATCHLIST 的股票）

觀察名單中，符合回調進場（路徑 A）或突破確認（路徑 B）才顯示：

| 判斷方式 | 條件 | 說明 |
|---|---|---|
| 路徑A 回調 | MA5>MA10、RSI 40~65、量比≥0.5、乖離≤5% | 標準回調進場 |
| 路徑B 突破 | MA5>MA10、RSI 50~75、量比≥1.5、日漲≥3% | 追強突破確認 |

### 區塊五：明日開盤操作清單

每項直接顯示建議掛單價格，開盤前看一眼就知道今天要做什麼：

| 類別 | 顯示內容 |
|---|---|
| 🔴 需要處理 | ATR 停損線（依獲利段自動計算），低於此價考慮出場 |
| 🟢 停利訊號 | 現價 + 建議減碼股數（持股的 30~50%） |
| 🟠 留意觀察 | 有技術面警示但不緊急，盤中留意 |
| 攤平候選 | 就緒時顯示 → 可在 {現價} 附近掛買單 |
| 建倉加碼 | 就緒時顯示 → 可在 MA5~MA5×0.98 範圍掛買單 |
| ✅ 正常續抱 | 無任何警示的持股 |

## 策略邏輯總覽

### 進場條件（新標的 / 觀察名單）

**路徑 A — 回調進場（推薦，風險報酬較佳）：**
```
MA5 > MA10（多頭排列）
AND RSI 40~65（有動能，不追高）
AND 量比 ≥ 0.5（量縮回調可接受）
AND 現價距 MA5 ≤ 5%（靠近支撐，不追漲）
```

**路徑 B — 突破確認（追強，適合強勢股）：**
```
MA5 > MA10
AND RSI 50~75
AND 量比 ≥ 1.5（放量確認）
AND 日漲幅 ≥ 3%（明確突破）
```

### 持倉加碼（building: True）

同上進場條件，路徑 A 或路徑 B 任一成立即可加碼。

### 持倉期間停利

```
乖離率 > 5%  且 RSI > 62 → 偏熱，留意（🟡）
乖離率 > 8%  且 RSI > 68 → 過熱，減碼 30%（🟢）
乖離率 > 12% 且 RSI > 75 → 強力停利，減碼 30~50%（🟢）
```

### 出場條件（任一觸發）

```
現價 < ATR 停損線（三段式移動停損）
單日跌幅 > 4%（跳空保護）
同時跌破 MA5 + MA10（趨勢反轉）
單獨跌破 MA10（中期支撐失守）
跌破 MA5 且 MACD 翻負（短線轉弱）
```

### ATR 停損線計算方式（三段）

```
獲利 < 10%  → 停損線 = 買入價 − 2×ATR        （保護本金）
獲利 10~20% → 停損線 = 買入價 + 獲利 × 40%   （保住部分獲利）
獲利 ≥ 20%  → 停損線 = 現價 − 2×ATR           （移動追蹤）
```

### 攤平條件（深套股，avg_down: True）

```
RSI < 60 且持續回升（止跌確認）
AND MA5 斜率翻正（短線止跌）
AND 今日收紅且量比 > 1.2（買盤進場）
AND 距近20日低點反彈 > 3%（底部確認）
（4個條件達到 3 個才執行）
```

## 技術指標說明

| 指標 | 計算方式 | 說明 |
|---|---|---|
| MA5 / MA10 | 5日/10日收盤均線 | 盤後以最終收盤重算，盤中保留前日值 |
| RSI(14) | **Wilder's SMMA**（EWM alpha=1/14） | 與 Yahoo Finance / TradingView 一致；注意：簡單 SMA 計算會偏低 10~20 點 |
| ATR(14) | 14日真實波動範圍均值 | 停損單位，越大代表波動越大 |
| MACD | EMA12 − EMA26，Signal=EMA9 | MACD_hist 翻正/負為方向訊號 |
| 量比 | 今日量 / 5日均量 | 盤中使用推估全日量（累計量÷已過進度） |
| 乖離率 | (現價 − MA5) / MA5 × 100% | 偏離均線程度，>5% 開始留意，>12% 強力警示 |

## 常見問題

**Q：為什麼有些股票顯示 N/A？**
A：部分股票（ETF、生技股等）在 TWSE API 可能無本益比資料，屬正常現象。

**Q：月營收顯示的是哪個月？**
A：FinMind 資料以「實際營收月份」標示，例如「2026-02」即 2 月份營收。
台股月營收每月 10 日公布上個月數字，10 日前僅能看到上上個月資料。

**Q：盤中跑和盤後跑有什麼差別？**
A：盤中執行時，MA5/MA10 保持前一日穩定值（避免盤中震盪造成誤判），量比為累計量推估全日。
收盤後執行，MA5/MA10 以 Fugle 最終收盤重算，指標精確對齊主流圖表軟體。

**Q：RSI 為什麼和之前不同？**
A：系統已改為 Wilder's SMMA（EWM alpha=1/14），與 Yahoo Finance、TradingView 等圖表軟體一致。
舊版使用簡單 SMA，計算結果會偏低 10~20 點，導致錯誤的超賣訊號。

**Q：ATR 停損線為什麼每次不一樣？**
A：停損線會依你的獲利狀況自動切換：
- 小獲利/虧損 → 固定在買入價下方（保護本金）
- 中等獲利 → 移至成本以上（保住 40% 獲利）
- 大幅獲利 → 跟著現價往上走（移動停損，鎖住利潤）

**Q：加碼後如何更新均價？**
A：開啟 `stocks.py`，更新對應股票的 `buy_price` 和 `shares`。
新均價 = (原均價 × 原股數 + 新成交價 × 新買股數) / 總股數
