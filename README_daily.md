## 設定持股（stocks.py）

打開 `stocks.py` 修改以下兩個區塊：

### 持倉設定

```python
HOLDINGS = {
    "2330.TW": {
        "name":      "台積電",        # 公司名稱（顯示用）
        "buy_price": 950.0,           # 平均買入價
        "shares":    1000,            # 持有股數
        "avg_down":  False,           # True = 深套，找低點攤平
        "building":  True,            # True = 建倉中，持續買進
    },
    "NVDA": {
        "name":      "輝達",
        "buy_price": 120.0,
        "shares":    10,
        "avg_down":  False,
        "building":  True,
    },
}
```

**avg_down 和 building 的差別：**

| 旗標 | 情境 | 程式行為 |
|---|---|---|
| `avg_down: True` | 深套，等反彈低接 | 偵測超賣反彈訊號（4個條件） |
| `building: True` | 持續建倉，順勢加碼 | 偵測順勢加碼時機（3個條件） |
| 兩個都 True | 深套但仍想持續加碼 | 同時顯示兩種訊號 |

### 觀察名單（還沒買的股票）

```python
WATCHLIST = {
    "tw": ["2330", "2317"],   # 台股代號（不用加 .TW）
    "us": ["NVDA", "AAPL"],   # 美股代號
}
```

### 更新持股均價（加碼後）

加碼後記得更新 `buy_price` 和 `shares`，例如：
```python
# 原本：100 股，均價 80 元
# 加碼：再買 100 股，成交 75 元
# 新均價：(80×100 + 75×100) / 200 = 77.5 元

"buy_price": 77.5,
"shares":    200,
```

## 報告說明

### 區塊一：持倉警示

每檔持股顯示現價、損益，以及以下警示：

| 符號 | 意義 | 建議動作 |
|---|---|---|
| 🔴 跌破 ATR 停損線 | 現價低於「買入價 - 2倍ATR」 | 評估是否出場 |
| 🔴 單日跌幅 > 4% | 當日跌幅超過 4% | 注意跳空風險 |
| 🟠 跌破 MA10 | 收盤價低於10日均線 | 注意風險 |
| 🟠 跌破 MA5 且 MACD 翻負 | 短線趨勢轉弱 | 考慮部分出場 |
| 🟡 跌破 MA5 但 MACD 仍正 | 可能只是正常回測 | 觀望 |
| 🟡 乖離率偏熱 | 偏離 MA5 > 5% 且 RSI > 65 | 留意是否減碼 |
| 🟢 乖離率 > 8% 且 RSI > 70 | 短線過熱 | 建議減碼 30% |
| 🟢 乖離率 > 12% 且 RSI > 78 | 強烈過熱 | 建議減碼 30% 或考慮出清 |
| ✅ 持倉正常 | 無任何警示 | 續抱 |

**基本面資料說明（近12個月滾動，僅供參考）：**
- 本益比、本淨比、殖利率：來自台灣證交所/櫃買中心官方 API，每日更新
- 月營收 YoY/MoM：來自 FinMind，為實際公布月份資料（台股月營收每月 10 日公布）

### 區塊二：攤平時機偵測（avg_down: True 的持股）

4個條件，達到 3 個以上顯示「攤平訊號確認」：

| 條件 | 說明 |
|---|---|
| RSI 從超賣區（<35）回升 | 賣壓釋放，開始有人接 |
| MA5 斜率翻正 | 短線止跌跡象 |
| 今日收紅且量比 > 1.2 | 有實際買盤進場 |
| 距近20日低點反彈 > 3% | 底部有支撐 |

### 區塊三：建倉加碼機會（building: True 的持股）

4個條件全達成才顯示「加碼時機」（回調加碼策略）：

| 條件 | 說明 |
|---|---|
| MA5 > MA10 | 多頭排列，方向對 |
| RSI 在 45~65 | 有動能但不追高 |
| 量比 > 0.8 | 至少不是極度量縮 |
| 現價距 MA5 不超過 5% | 回調加碼，不追高 |

### 區塊四：明日進場機會（WATCHLIST 的股票）

觀察名單中，符合以下 3 個條件才顯示：

| 條件 | 說明 |
|---|---|
| MA5 > MA10 | 多頭排列 |
| RSI > 50 | 動能向上 |
| 量比 > 1.2 | 有量配合 |

### 區塊五：明日開盤操作清單

每項直接顯示建議掛單價格，開盤前看一眼就知道今天要做什麼：

| 類別 | 顯示內容 |
|---|---|
| 🔴 需要處理 | 停損參考價（買入價 - 2×ATR），低於此價考慮出場 |
| 🟢 停利訊號 | 現價 + 建議減碼股數（持股的 30%） |
| 🟠 留意觀察 | 有技術面警示但不緊急，盤中留意 |
| 攤平候選 | 就緒時顯示 → 可在 {現價} 附近掛買單 |
| 建倉加碼 | 就緒時顯示 → 可在 MA5~MA5×0.98 範圍掛買單 |
| ✅ 正常續抱 | 無任何警示的持股 |

## 策略邏輯總覽

### 進場條件（建倉 / 新標的）
```
MA5 > MA10（多頭排列）
AND RSI 40~65（有動能不追高）
AND 現價距 MA5 不超過 5%
```

### 持倉期間
```
跌破 MA5 但 MACD 仍正 → 觀望，不出場
乖離率 > 8% 且 RSI > 70 → 考慮減碼 30%
乖離率 > 12% 且 RSI > 78 → 考慮大幅減碼或出清
```

### 出場條件（任一觸發）
```
現價 < 買入價 - 2×ATR（動態停損）
單日跌幅 > 4%（跳空保護）
收盤跌破 MA10（趨勢轉壞）
跌破 MA5 且 MACD 翻負（短線轉弱）
```

### 攤平條件（深套股）
```
RSI 從超賣區回升（近10日曾 < 35，現在 > 35）
AND MA5 斜率翻正
AND 今日收紅且量比 > 1.2
AND 距近20日低點反彈 > 3%
（4個條件達到 3 個才執行）
```

## 常見問題

**Q：為什麼有些股票顯示 N/A？**
A：部分股票（ETF、生技股等）在 TWSE API 可能無本益比資料，屬正常現象。

**Q：月營收顯示的是哪個月？**
A：FinMind 資料以「實際營收月份」標示，例如「2026-02」即 2 月份營收。
台股月營收每月 10 日公布上個月數字，10 日前僅能看到上上個月資料。

**Q：盤中跑和盤後跑有什麼差別？**
A：技術指標（MA、RSI、ATR）均以收盤價計算，盤中執行顯示的是「最新成交價 + 昨日指標」。
成交量在盤中為累計值，尚未完整，量比僅供參考。

**Q：加碼後如何更新均價？**
A：開啟 `stocks.py`，更新對應股票的 `buy_price` 和 `shares`。
新均價 = (原均價 × 原股數 + 新成交價 × 新買股數) / 總股數
